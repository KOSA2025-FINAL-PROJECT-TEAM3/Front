# [Feat] 질병 검색 UI 및 등록 프로세스 개선

## ✨ PR 요약
질병 검색 탭의 UX를 개편하여 검색에서 등록으로 이어지는 흐름을 매끄럽게 개선했습니다. '검색' 버튼을 '등록'으로 변경하여 질병 관리 페이지와의 연동성을 강화하였으며, AI 및 Google 검색 기능을 추가하여 정보 탐색 편의성을 높였습니다.

## 🔗 관련 이슈
Closes #146

## 🛠️ 변경 내용

### Frontend
*   **DiseaseSearchTab.jsx**:
    *   `검색` 버튼 -> `등록` 버튼 변경: 버튼 클릭 시 질병 관리 페이지(`/disease`)로 이동하며, 입력한 키워드가 **질병 등록 폼에 자동 입력(Pre-fill)**됩니다.
    *   `Google 검색` 버튼 추가: 클릭 시 새 탭에서 구글 검색 결과를 보여줍니다.
    *   `AI 검색` 버튼 및 패널 추가: 클릭 시 페이지 이동 없이 **모달 우측 패널**에 AI 분석 결과(요약, 증상 등)를 즉시 표시합니다. (현재는 Mock Data 적용)
    *   문구 수정: "질병 이름이나 증상을 검색하세요"로 변경.
*   **DiseasePage.jsx**: 
    *   `location.state.autoCreate` 값을 감지하여, 페이지 진입 시 자동으로 **질병 등록 모달**을 열고 병명을 입력해 주는 로직을 추가했습니다.
*   **SearchOverlay.jsx**:
    *   `DiseaseSearchTab` 컴포넌트 연동 및 이전에 발생했던 문법 오류(Malformed JSX)를 수정/복구했습니다.
*   **searchOverlayStore.js**:
    *   `disease` 탭 활성화를 막고 있던 정규화 로직(`normalizeTab`)을 수정하여 정상적으로 탭 전환이 되도록 했습니다.
*   **SeniorDashboard.jsx**:
    *   대시보드의 '질병 검색' 버튼이 오버레이 모달을 열도록 연결(`openSearchOverlay('disease')`)했습니다.

## 🙋 리뷰어에게
*   **AI 검색 기능**: 현재는 백엔드 API가 준비되지 않아 프론트엔드에서 Mock Data(가상 데이터)로 동작하도록 구현했습니다. 향후 백엔드 API가 개발되면 `diseaseApiClient`를 업데이트하여 연동해야 합니다.
*   **검증 방법**:
    1.  대시보드에서 '질병 검색' 클릭 -> 오버레이 열림 확인.
    2.  검색창에 병명(예: 고혈압) 입력 후 '등록' 클릭 -> 질병 관리 페이지로 이동하며 등록 폼이 열리는지 확인.
    3.  'AI 검색' 클릭 -> 우측 패널에 결과 표시 확인.
