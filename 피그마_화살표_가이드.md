# 🎯 피그마에서 화살표로 연결된 와이어프레임 보는 방법

## 핵심 포인트

**PNG 이미지 (`noarro.png`)는 정적이라서 화살표가 안 보입니다!**

화살표를 보려면 → **피그마 플러그인으로 JSON을 import**해야 합니다.

---

## 📋 준비물

✅ `all-screens-with-flows.json` - 화살표 연결 정보 포함
✅ `figma-plugin/` - 이미 컴파일 완료
✅ 피그마 Desktop 앱 (웹 버전 안됨!)

---

## 🚀 3단계로 화살표 보기

### 1단계: 피그마에 플러그인 설치 (최초 1회)

1. **피그마 Desktop 앱** 열기
2. 메뉴: `Plugins → Development → Import plugin from manifest...`
3. 이 프로젝트의 `figma-plugin/manifest.json` 선택
4. ✅ 플러그인이 개발 플러그인 목록에 추가됨

---

### 2단계: JSON 파일 복사

```bash
# 프로젝트 폴더에서
cat figma-exports/all-screens-with-flows.json
```

**전체 내용을 복사** (Ctrl+A → Ctrl+C)

---

### 3단계: 피그마에서 Import

1. 피그마에서 **새 파일** 생성 (또는 기존 파일 열기)
2. 메뉴: `Plugins → Development → 실버케어 JSON Importer`
3. 플러그인 UI에 **JSON 붙여넣기** (Ctrl+V)
4. **🚀 Import Multiple** 버튼 클릭
5. ⏳ 잠깐 기다리면...

---

## ✨ 결과

- 약 관리 → 홈
- 가족 관리 → 홈

## 🎨 화살표 확인하는 법

### 방법 1: 프로토타입 모드

1. 피그마 오른쪽 상단에서 **프로토타입 탭** 클릭
2. 화면을 선택하면 → 연결된 화살표가 보임!
3. 화살표를 클릭하면 어디로 연결되는지 확인 가능

### 방법 2: 프레젠테이션 모드

1. 화면 선택 후 오른쪽 상단 **▶️ 재생 버튼** 클릭
2. 실제로 클릭하면서 화면 전환 테스트 가능
3. ESC로 나가기

---

## 🔧 문제 해결

### "Plugin not found"
→ 피그마 **Desktop 앱**을 사용하고 있는지 확인 (웹 버전 안됨)

### "Font not available" 경고
→ [Inter 폰트](https://fonts.google.com/specimen/Inter) 설치 필요
→ 또는 피그마에서 다른 폰트로 변경

### "Invalid JSON structure"
→ JSON 파일 전체를 복사했는지 확인
→ 파일 시작: `{` / 파일 끝: `}`

### 화살표가 안 보임
→ **프로토타입 탭**에서 확인 (디자인 탭에서는 안 보임)
→ 화면 선택 후 오른쪽 패널의 "Prototype" 클릭

## 💡 Tip

- 화살표는 **DISSOLVE** 트랜지션 (0.3초)으로 설정되어 있음
- 프로토타입 모드에서 화살표 설정 변경 가능
- 각 화면은 **Auto Layout** 적용되어 있어 수정 가능

---

## 📸 PNG vs 피그마 플러그인

| 구분 | PNG 이미지 | 피그마 플러그인 |
|------|------------|----------------|
| 화살표 | ❌ 없음 | ✅ 있음 |
| 수정 가능 | ❌ 불가 | ✅ 가능 |
| Auto Layout | ❌ 없음 | ✅ 적용됨 |
| 인터랙션 | ❌ 없음 | ✅ 클릭 가능 |

**결론: PNG는 정적 이미지일 뿐, 피그마 플러그인을 사용해야 화살표를 볼 수 있습니다!**
